---
title: "Song map (ISongMap)"
description: "The structured musical analysis of a song, providing beats, chords, and repetitive segments from Songle."
---

`ISongMap` contains the full musical analysis of a loaded song, generated by [Songle](https://songle.jp/). Once a video is ready, you can access the song map through `player.data.songMap` to retrieve beat grid, chord progressions, and structurally repeated sections such as choruses.

<Note>
  The song map is available after the `onVideoReady` callback fires. Do not access `player.data.songMap` before that point — it will be `null`.
</Note>

## Accessing the song map

```typescript
const player = new Player({
  app: { token: "your-token" },
});

player.addListener({
  onVideoReady(video) {
    const songMap = player.data.songMap;

    console.log("Beats:", songMap.beats.length);
    console.log("Chords:", songMap.chords.length);
    console.log("Segment groups:", songMap.segments.length);
  },
});
```

## ISongMap properties

<ResponseField name="beats" type="IBeat[]">
  Array of all beats detected in the song, sorted by `startTime`. Each element is an `IBeat` object.

  See [Beat (IBeat)](/api/beat) for full property reference.
</ResponseField>

<ResponseField name="chords" type="IChord[]">
  Array of all chord progressions in the song, sorted by `startTime`. Each element is an `IChord` object.

  See [Chord (IChord)](/api/chord) for full property reference.
</ResponseField>

<ResponseField name="segments" type="IRepetitiveSegments[]">
  Array of repetitive segment groups (e.g., chorus sections). Each `IRepetitiveSegments` element groups multiple individual `IRepetitiveSegment` objects of the same structural type.

  See [Repetitive segments](/api/repetitive-segment) for full property reference.
</ResponseField>

<ResponseField name="revisions" type="object">
  Revision IDs for the song map data. These values reflect which specific analysis version was loaded, and can be used to pin a song to a particular analysis revision when calling `createFromSongUrl`.

  <Expandable title="properties">
    <ResponseField name="chordId" type="number | undefined">
      Revision ID for the chord analysis data.
    </ResponseField>

    <ResponseField name="beatId" type="number | undefined">
      Revision ID for the beat analysis data.
    </ResponseField>

    <ResponseField name="repetitiveSegmentId" type="number | undefined">
      Revision ID for the repetitive segment analysis data.
    </ResponseField>
  </Expandable>
</ResponseField>

  <Expandable title="properties">

    <ResponseField name="name" type="string">
      Song title.
    </ResponseField>

    <ResponseField name="length" type="number">

    <ResponseField name="artist" type="object">

<ResponseField name="songMap" type="ISongMap">

## Overriding song map data with PartialVideoEntry

You can override which analysis revision is used when loading a song by passing a `video` option to `createFromSongUrl`. This is done via the `PartialVideoEntry` interface.

```typescript
player.createFromSongUrl("https://www.youtube.com/watch?v=example", {
  video: {
    // Use a specific lyrics timing ID (-1 = latest, 0 = none)
    lyricId: -1,
    lyricDiffId: 0,

    // Pin the song map to specific analysis revisions
    beatId: 3,
    chordId: 2,
    repetitiveSegmentId: 1,
  },
});
```

<ResponseField name="lyricId" type="number | undefined">
  Specifies which lyrics timing data to load.
  - `-1`: Use the latest available lyrics info
  - `0`: Do not load any lyrics info
  - Any other positive integer: Load the specified lyrics ID
</ResponseField>

<ResponseField name="lyricDiffId" type="number | undefined">
  Revision ID for lyrics corrections (diff).
</ResponseField>

<ResponseField name="chordId" type="number | undefined">
  Revision ID for chord analysis data.
</ResponseField>

<ResponseField name="beatId" type="number | undefined">
  Revision ID for beat analysis data.
</ResponseField>

<ResponseField name="repetitiveSegmentId" type="number | undefined">
  Revision ID for repetitive segment analysis data.
</ResponseField>

<ResponseField name="json" type="VideoData | undefined">
  Full video data to embed directly, bypassing the server-side video fetch.
</ResponseField>

## TimedObjectsInRange\<T\>

All `findXxxChange` methods on `player` return a `TimedObjectsInRange<T>` result. This structure describes what happened between the last frame and the current one:

```typescript
interface TimedObjectsInRange<T extends TimedObject> {
  // The object that overlaps the end time of the queried range (currently active)
  current: T | null;
  // Objects that started within the queried range
  entered: T[];
  // Objects that ended within the queried range
  left: T[];
  // The last object before the queried range
  previous: T | null;
  // The first object after the queried range
  next: T | null;
}
```

### Usage example

```typescript
player.addListener({
  onTimeUpdate(position) {
    const beatChange = player.findBeatChange(
      player.videoPosition,
      position
    );

    for (const beat of beatChange.entered) {
      console.log("New beat entered at", beat.startTime);
    }
  },
});
```

<Tip>
  Use `entered` to detect when a new beat, chord, or segment has just become active. Use `current` to check which object is active at the current moment.
</Tip>
